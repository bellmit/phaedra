Normalization Methods

In Phaedra, well feature values can be normalized. They are transformed into values that are relative to the plate's control wells, and allow easier comparison against values from other plates.

Normalization methods are specified in the protocol class editor, by selecting a feature and selecting a normalization in the list of available methods.


The predefined methods that are available in Phaedra are listed below.

Method Formula
%Effect ( rawValue - lowMedian ) / ( highMedian - lowMedian ) * 100
%Effect(100-x) 100 - %Effect
%Effect(-x) - %Effect
%Effect(x-100) %Effect - 100
Sigma[H] ( rawValue - highMedian ) / highStdDev
Sigma[L] ( raw - lowMedian ) / lowStdDev
%CTLMin %Effect
%CTL rawValue / highMedian * 100
%CTL[L] ( ( rawValue - lowMedian ) / lowMedian ) * 100
%CTL[L=0] %CTL[L]
%CTL[INV L=0] - %CTL[L]
%CTL[L=100] ( rawValue - lowMedian ) * 100
%CTL[H=0] ( ( highMedian - rawValue ) / highMedian ) * 100
PIN[Pos Mean] (rawValue - lowMean) / (highMean - lowMean) * 100
PIN[Pos Median] (rawValue - lowMedian) / (highMedian - lowMedian) * 100
PIN[Neg Mean] 100 - (rawValue - lowMean) / (highMean - lowMean) * 100 = 100 - PIN[Pos Mean]
PIN[Neg Median] 100 - (rawValue - lowMedian) / (highMedian - lowMedian) * 100 = 100 - PIN[Pos Median]
ZScore (rawValue - mean) / sd
ZScore[L] (rawValue - lowMean) / lowSd
ZScore[Rob] (rawValue - median) / (1.4826 * mad)
ZScore[Rob L] (rawValue - lowMedian) / (1.4826 * lowMad)
Note: While custom normalization methods are supported, they should be used with great care. Contact a Phaedra administrator before setting a custom normalization formula.
