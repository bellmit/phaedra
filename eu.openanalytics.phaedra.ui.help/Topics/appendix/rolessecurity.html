Roles and Security

In Phaedra, each protocol is owned by a team. If you are not a member of the owning team, you cannot access the contents of that protocol.

Within a team, there are four possible membership levels:


  • Read-only user: can view all data, but cannot modify anything.
  • User: can create, modify and delete plates. Can create, modify and delete experiments. Can perform plate validation and trigger plate recalculations. If the protocol is in development, the user can also modify the protocol.
  • Manager: As User, plus the permission to perform plate approval. Can modify a protocol, even if it is not in development. Can create, modify and delete protocols.
  • Administrator: Full adminstrative access.
